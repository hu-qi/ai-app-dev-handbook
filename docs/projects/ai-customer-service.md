# 项目二：AI 客服机器人

AI 客服机器人适合电商、SaaS、教育、企业服务等场景。它不是简单聊天机器人，而是知识库、意图识别、订单工具、工单系统和人工接管的组合。

## 一、项目目标

构建一个能回答常见问题、查询订单状态、识别复杂问题并转人工的客服助手。

## 二、核心模块

- FAQ 问答
- 知识库检索
- 用户意图识别
- 订单查询工具
- 退款规则判断
- 工单创建
- 人工接管
- 会话摘要
- 质检分析

## 三、典型流程

```text
用户提问
  -> 意图识别
  -> FAQ / RAG / 工具调用
  -> 生成回复
  -> 判断是否需要人工
  -> 保存会话与质检标签
```

## 四、意图分类

常见意图：

- 售前咨询
- 订单查询
- 物流查询
- 退换货
- 投诉
- 发票
- 人工客服
- 无法识别

意图识别可以先用规则 + 小模型完成，复杂问题再交给大模型。

## 五、工具设计

### getOrderStatus

输入：orderId

输出：订单状态、物流单号、预计到达时间。

### createTicket

输入：用户信息、问题摘要、优先级。

输出：工单编号。

### checkRefundPolicy

输入：订单信息、退款原因。

输出：是否符合退款条件、原因说明。

## 六、人工接管策略

以下情况必须转人工：

- 用户强烈投诉
- 涉及退款争议
- 模型无法确认答案
- 用户连续两次不满意
- 涉及敏感信息或高风险操作

## 七、质检与运营

每次会话都应该沉淀为运营数据：

- 用户问题类型
- 是否解决
- 是否转人工
- 用户满意度
- 高频知识缺口
- 客服节省时长

## 八、验收标准

- FAQ 命中率可统计
- 订单查询工具可用
- 复杂问题能转人工
- 会话有摘要和标签
- 高风险动作不会自动执行
