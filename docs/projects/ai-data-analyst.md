# 项目三：AI 数据分析助手

AI 数据分析助手的价值是让业务人员用自然语言理解数据、提出问题、生成图表和发现异常。

## 一、项目目标

构建一个支持上传表格、理解字段、回答数据问题、生成分析结论的数据助手。

## 二、核心能力

- 表格上传
- 字段识别
- 数据质量检查
- 自然语言转 SQL
- 图表建议
- 异常识别
- 分析报告生成
- 结论可追溯

## 三、架构流程

```text
上传数据
  -> 字段解析
  -> 数据入库
  -> 用户提问
  -> 生成 SQL
  -> 安全校验
  -> 执行查询
  -> 生成图表和结论
```

## 四、SQL 安全

不要让模型直接执行任意 SQL。必须限制：

- 只允许 SELECT
- 禁止 DROP / DELETE / UPDATE
- 限制查询行数
- 限制执行时间
- 对字段名做白名单校验

## 五、字段语义层

业务数据通常字段名不友好。建议建立语义层：

- 字段英文名
- 字段中文名
- 字段含义
- 数据类型
- 统计口径
- 示例值

模型应该基于语义层生成查询，而不是猜字段含义。

## 六、分析输出

一个好的数据回答应该包含：

1. 直接结论
2. 核心数据
3. 对比关系
4. 可能原因
5. 下一步建议
6. 查询口径说明

## 七、适用场景

- 销售数据分析
- 广告投放复盘
- 库存周转分析
- 客服问题统计
- 财务费用分析
- 用户增长分析

## 八、验收标准

- SQL 不越权
- 结论和数据一致
- 图表类型合理
- 能说明统计口径
- 能识别异常但不夸大原因
